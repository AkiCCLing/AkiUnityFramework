# UIFrame 自动绑定 - 快速参考卡

## 🚀 快速开始

### 方式1：创建新脚本时自动生成绑定脚本
```
1. 在Prefab上右键
2. 选择 Assets/Create/UIFrame/UIPanel（或 UIWindow/UIComponent/UIBase）
3. ✅ 同时生成主脚本和Binding脚本
4. ✅ 脚本组件自动添加到Prefab根节点
```

### 方式2：为现有Prefab生成/更新绑定脚本
```
1. 在Prefab上右键
2. 选择 Assets/UIFrame/重新生成绑定脚本
3. ✅ 更新 *.Binding.cs 文件
4. ✅ 确保脚本组件已添加到Prefab
```

## 📝 命名规则速查表

| 前缀 | 类型 | 字段类型 | 示例 |
|------|------|---------|------|
| `@txt_` | 文本 | `TextMeshProUGUI` | `@txt_Title` → `txtTitle` |
| `@img_` | 图片 | `Image` | `@img_Icon` → `imgIcon` |
| `@btn_` | 按钮 | `Button` | `@btn_Close` → `btnClose` |
| `@input_` | 输入框 | `InputField` | `@input_UserName` → `inputUserName` |
| `@raw_` | 原始图片 | `RawImage` | `@raw_Background` → `rawBackground` |
| `@` | 物体 | `GameObject` | `@Content` → `content` |

## 💡 使用示例

### 创建Prefab结构
```
MyLoginPanel (Prefab)
├── @txt_Username
│   └── TextMeshProUGUI
├── @txt_Password
│   └── TextMeshProUGUI
├── @img_Logo
│   └── Image
├── @btn_Login
│   └── Button
└── @btn_Register
    └── Button
```

### 生成的字段声明（MyLoginPanel.Binding.cs）
```csharp
#region Field Declarations
[SerializeField] private TextMeshProUGUI txtUsername;
[SerializeField] private TextMeshProUGUI txtPassword;
[SerializeField] private Image imgLogo;
[SerializeField] private Button btnLogin;
[SerializeField] private Button btnRegister;
#endregion
```

### 生成的绑定代码（MyLoginPanel.Binding.cs）
```csharp
private void BindReferences()
{
    txtUsername = transform.Find("@txt_Username").GetComponent<TextMeshProUGUI>();
    txtPassword = transform.Find("@txt_Password").GetComponent<TextMeshProUGUI>();
    imgLogo = transform.Find("@img_Logo").GetComponent<Image>();
    btnLogin = transform.Find("@btn_Login").GetComponent<Button>();
    btnRegister = transform.Find("@btn_Register").GetComponent<Button>();
}
```

## ❓ 常见问题

**Q: Binding.cs 文件能否手动编辑？**
> A: 不建议。Auto Bind References 区域在重新生成时会被覆盖。如需自定义逻辑，在主脚本的 Private Functions 区域添加。

**Q: 如何调用绑定？**
> A: 在 OnBind() 或 OnCreate() 中调用 `BindReferences()`。

**Q: 不想用某个 @ 组件？**
> A: 移除 @ 前缀即可，该组件不会被绑定。

**Q: 支持嵌套Prefab吗？**
> A: 支持。BreadthTraversal 会遍历所有子物体。

**Q: 修改前缀前的字段名后需要重新生成吗？**
> A: 是的。重新生成后 Binding.cs 会使用新名称。

## 🔧 配置注意事项

- **Binding.cs 不应添加到版本控制**（可添加到 .gitignore）
- **主脚本必须为 partial class**（模板已默认设置）
- **GameObject 名称必须唯一**（同一层级不能重复）
- **确保使用了正确的 using 语句**（Binding.cs 自动添加）
